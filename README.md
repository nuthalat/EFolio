# Efolio — A Curated Engineering Portfolio

## Overview

This repository contains my personal portfolio website—a structured, first-principles representation of my professional work as a software engineer.

While resumes provide a compressed view of experience, this portfolio is designed to offer a clearer perspective on the systems I’ve built, the domains I’ve worked in, and the scale at which I’ve operated.

## Why this portfolio exists

Over the past **9+ years**, I’ve worked on large-scale systems across cloud infrastructure, data platforms, enterprise integrations, and media systems—primarily within **Amazon**.

Much of this work is not publicly visible.

This portfolio exists to:

- Provide a transparent view of my experience beyond a resume
- Highlight the domains and systems I’ve worked on
- Present my work in a structured, accessible format

## What this repository contains

This repository powers a resume-style portfolio website, which includes:

- Professional summary and experience
- Key projects and system contributions
- Technical skills and areas of expertise
- Domain exposure across multiple large-scale systems

The goal is to present information in a way that is:

- Clear and navigable
- Focused on impact and scale
- Easy to evaluate for technical audiences

## Design philosophy

The portfolio is intentionally minimal.

I’ve optimized for:

- Clarity over visual complexity
- Signal over noise
- Structured presentation of experience

The goal is not to showcase frontend complexity, but to make the content easy to consume and evaluate.

## On portfolios in the age of AI

The role of portfolios is evolving.

AI can now generate:

- Websites
- UI components
- Boilerplate project structures

As a result, the value of a portfolio is no longer in how it is built, but in what it represents.

A meaningful portfolio should:

- Clearly communicate real-world experience
- Reflect depth in system design and engineering decisions
- Provide context around impact and scale

This repository is built with that perspective.

## Who this is for

- Recruiters evaluating technical experience
- Hiring managers assessing system-level impact
- Engineers looking to understand my background and areas of expertise

## Preview the website locally

From the project folder, run:

```bash
python3 -m http.server 8000
```

Then open `http://localhost:8000`.

- `index.html` — page structure and content
